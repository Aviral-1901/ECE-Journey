# Day 14 : Makefile

## What problem Makefile solves
To test a unit we had to manually type "iverilog -g2012 -o ..._sim ....v ..._tb.v  \ vvp .._sim" and for testing 50 units we have to manually type the 2 commands 50times and have to remember which unit depends on which other units. So using makefile we solve this manual typing when we need to test the unit and dependency problem is solved.

## Makefile rule structure
A rule has three parts:
- target : the name we call with make
- dependencies : other rules that must run first
- commands : what to execute (must use TAB)
Structure : <br>
target dependencies <br>
-> TAB command1
-> TAB command2


## Variables
Using variables $() we can change the path at first and everything changes accordingly. No hardcoded paths.

## Dependency chains
full_adder.v internally instantiates half_adder. When iverilog compiles full_adder_tb it needs to find half_adder module definition. So we include half_adder.v in the compile command.

## make test
By typing make test command. Every unit is self tested at once and no need to manually type everything and allows batch testing.
