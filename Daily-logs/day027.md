# Day 27 : RISC-V Instruction Encoding (Part-1)

## Why encoding
CPU fetches 32 bit instructions from the memory. Decoder decodes those 32bits to extract meaning like what type of instruction is this, what it does and so on.

## Opcode
Opcode is 7bits. It determines the type of instruction.

## R-type
It is register-to-register type of instruction. We have 2 source registers and 1 destination register. It looks like:<br>
[funct7] [rs2] [rs1] [funct3] [rd] [opcode] <br>
- Opcode[6:0] = 0110011 for R-type instructions
- rd[11:7] = destination register (registers represented by 5bits binary)
- funct3[14:12]
- rs1[19:15] = source register 1
- rs2[24:20] = source register 2
- funct7[31:25] 
Opcode is same for one type of instruction. But there are many operations inside R-type instruction so funct3 and funct7 are required to know what specific operation is this inside the R-type instruction.

## I-type
It uses an immediate value. There is a constant number placed inside the instruction itself.<br>
[imm[11:0]] [rs1] [funct3] [rd] [opcode]<br>
- opcode[6:0] = 0010011 for arithmetic, 0000011 for loads
- rd[11:7] = destination register
- funct3[14:12]
- rs1[19:15] = source register 1
- 12-bits immediate value [31:20] = the immediate value that is in the instruction

## S-type
It is a store instruction. In this type of instruction one register has the value to be stored and one register contains the base address of the memory. By adding immediate to the base address we get the memory address we need.<br>
[imm[11:5]] [rs2] [rs1] [funct3] [imm[4:0]] [opcode] <br>
- opcode[6:0] = 0100011
- imm[4:0] in [11:7] = lower 5 bits of the immediate
- funct3[14:12]
- rs1[19:15] = base address register
- rs2[24:20] = register whose value is being stored
- imm[11:5] in [31:25] = upper 7 bits of the immediate

## Hardware Insight
In all 3 types of instruction rs1 is always in bits 19:15 and rs2 is always in bits 24:20. So the decoder can always find these bits in same place. If they were in different position then we would need mux and other bigger circuits to extract the bits. So this is the exact reason why immediate values in S-type are split to keep this positions of registers fixed.
