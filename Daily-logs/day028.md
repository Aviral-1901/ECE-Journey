## Negative Immediate Encoding
-1 and -4 were the negative immediate encoding used. They are not something different. We use 2's complement to get the -ve numbers and for making the numbers satisfy the number of bits we fill the remaining bits with 1.

## Sign Extension
The sign is extended based on the MSB like if MSB is 1 then 1 is filled up and if MSB is 0 then 0 is filled up to make smaller numbered bit to higher one like filling the rest of numbers of MSB to make 12 bit value to 32 bit value. In hardware sign Extension requires zero logic gates and takes zero time. It is achieved through routing wires (taking the MSB wire and fanning them to all other upper bit-lines)

## How RISC-V handles all large constants
Every numbers have to go through registers. ALU cannot write directly to memory or cannot load directly from memory without using registers. Immediate value means the values which can be included directly in the instruction as constants. So the maximum limit for immediate is 12 bits which is -2048 to 2047. So for importing data from RAM full 32 bit values we can load with lw or use LUI + ADDI to build large constants. The 12-bit immediate limit is only for how much big of a constant we can write directly in code, we can easily store or use full 32 bit values without any problem.
