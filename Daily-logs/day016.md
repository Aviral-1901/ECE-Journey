# Day 16 : always_comb, Packed Structs

## always_comb, always_ff, always @(*)
always @(sensitivity list) here we have to declare every signals that could change the output, missing one would cause the system to work unreliably. Then to solve this always @(*) was introduced which handled the problem of missing signals but the simulator didn't know the intent so it may create latches(memory) when trying to create combinational logic. To solve these we got always_comb which strictly means combinational logic and every signal should be assigned. If a signal is not assigned in every branch inside always_comb, the tool throws a compile error catching the bug immediately rather than placing a latch. always_ff is for sequential logic and they are sensitive to clock and uses non-blocking assignments(<=).


## Packed Structs
packed struct means all the fields are stored as continuous sequence of bits in memory with no gaps between them.All bits stored together as one larger-bit value. Using one output port (like in adder_subtractor ) helps manage the code and avoids being messy on using many different signals.
