# Day 25 : Synchronous RAM Build

## Parameters
Parameters allow us to use the module wherever needed just by changing the parameters. We can include parameters in the module declaration part using #() and bits to represent address locations changes with depth of RAM so we can use $clog2(DEPTH) for dynamic address bits handling.
We can write once and configure at instantiation for different designs.

## Single port RAM
The ram we built is single port RAM which means we have only one port for either reading or writing. We can only either read or write in one clock cycle but not both. So there is no external hazard in this structure.


## Forwarding
Forwarding was necessary if for one clock cycle read and write address were the same. In this case the write data would directly go to the output for bypassing memory read. Our single-port RAM has only one address for both to either read or write. The hazard problem is not there and also without forwarding everything would work well. But as a good practice and to handle any off cases at any instant we kept the forwarding step.
