## decoder.py
In decoder.py we give it the hex value of the instruction, then it decodes the bits and tells what type of instruction is that. There are different conditions for opcode and based on that we know the type of instruction and extract required bits for the corresponding position.

## Sign extension
The immediate value is a 12-bit binary value that can be positive or negative. However, when we extract it Python treats it as a standard unsigned binary number (from 0 to 4095) rather than 2's complement. Because the 12th bit is the sign bit, any extracted value that is 2048 or greater means it was originally a negative number. To fix this in Python, we subtract 4096 (2^12) from those large numbers to get the correct negative decimal value.

## S-type immediate construction
We left shift the imm_upper by 5 so that we can make space for lower 5 bits to construct the full immediate value.
