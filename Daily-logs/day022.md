# Day 22 : Register File Architecture

## Register File 
Register File is a combination of registers and circuitry to read and write. For our base RISC-V we have 32 registers(x0 to x31) each 32bits wide. The register file has 2 read ports and 1 write ports. So it is like 32 registers and we have 2 32:1muxes to select 2 register for reading at a time. And for writing we have a decoder that takes 5bit address and activates one of the register. 


## Why 2R1W
Most of the risc-v instructions need 2reads and 1 write so designing this in the hardware makes the system small, fast and power-efficient.

## X0 hardware
Among the 32 registers, register x0 is hardwired to 0. Reading it always gives 0 and writing to it is ignored and is 0 as well. RISC-V doesn't have in built mov instruction. The data is moved by adding contents of one register with x0. We can use the mv instruction for moving data but behind the scenes the assembler converts this like addi x1, x2, x0. 


## New things that was interesting to know
Register files having ports and we are able to read and write only at certain registers using the ports was interesting thing to learn. At first the x0 register always giving 0 was making me feel we wasted a whole register for nothing. But came to know later that this offered very interesting features like using addition with this for moving data.
