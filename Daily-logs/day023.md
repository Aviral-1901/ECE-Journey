# Day 23 : Register File Build

## Key design decisions
Reads are in always_comb block and writes are in always_ff blocks. So now in one cycle we can read, calculate the values and write in the register before the edge arrives. If reads were sequential then it would require one clock cycle to read data from, one to calculate and write in the registers. So due to this everything becomes one clock-cycle late. Using combinational reads we can make single cycle execution possible.

## x0 insight
writes to x0 are ignored because reading from x0 always gives 0. But checks are added in always_ff block as well so that we dont write data in x0. Its for clarity and not strictly required because if we have written on x0 as well then we would have read 0 from it always.
