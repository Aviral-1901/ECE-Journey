# Day 13 : Git Workflow and Repo Structure

## git add -p
Using this we can selectively commit the part that we want. When we make a change in a file it gets divided into hunks. The changes that are near gets in one hunk and changes that are far gets in different hunk. In diff + means the line that was added and - means the line was removed. Using this we can selectively stage the changes so that we can see how the parts of file has changed over different changes and its better than git add . because it commits everything at once and we dont clearly know what was the specific change made.


## git commit --amend
It is used when there is something wrong/typo in the last commit message. It can only be used before pushing the code. We can also add forgotten files to the last commit.


## git rebase -i
It is used to edit commit history.<br>
pick -> keep commit as it is <br>
r -> reword - change the commit message only <br>
s -> squash - combine with previous, keep both messages <br>
d -> drop - delete the commit entirely <br>
Never rebase pushed commits that is already shared collaboratively.
HEAD ~3 means go to the 3 commits before the current one.
